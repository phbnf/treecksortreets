How to Play

Merkle Trees use specific coordinates and nomenclature to trace hierarchy.

Nomenclature:
• Root Node: The top hash representing the whole tree.
• Leaf Node: A bottom data record (level 0).
• Sibling: The matching node paired at the same height level.
• Parent: The intermediate node computed directly above two children.

The root hash mismatch indicates that one transaction was tampered with.

Scan: Click nodes to check branch integrity.
🔴 Red: Culprit leaf is down this sub-branch.
🟢 Green: Sub-branch matches expectations.
Goal: Scan optimally in log2(N) steps (3 scans for a tree of size 8), click the culprit leaf, and declare it.

Instead of a magic scanner, audit the actual SHA-256 hash chains!

Click a parent node to load its child hashes in the Verification Console below.
